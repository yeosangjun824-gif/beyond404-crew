현재 웹앱에서 카카오맵을 사용해 크루원의 실시간 위치를 표시하고 있다.
사용자의 위치는 고정이고, 크루원의 위치가 주기적으로 서버/클라이언트로 전달된다.

현재 문제:

* accuracy 필터만 적용했지만 여전히 약 60m 정도 위치가 튄다.
* 단순히 좌표를 그대로 마커에 표시하면 서비스 품질이 떨어진다.
* 따라서 좌표 수신 → 필터링 → 점프 제거 → smoothing → 경로/도로 스냅 → 마커 애니메이션까지 하나의 위치 보정 파이프라인으로 구현하고 싶다.

목표:

1. accuracy 필터
2. 최근 좌표 가중 평균
3. 비정상 점프 제거
4. smoothing / Kalman 유사 필터
5. route snap 또는 map matching
6. 카카오맵 마커 부드러운 이동 애니메이션

을 현재 코드 구조에 맞게 적용해라.

구현 요구사항:

## 1. 위치 데이터 타입 정리

크루원 위치 객체는 아래 정보를 기준으로 처리한다.

```ts
type CrewLocation = {
  crewId: string;
  lat: number;
  lng: number;
  accuracy?: number; // meters
  speed?: number;    // m/s
  heading?: number;  // degrees
  timestamp: number; // ms
};
```

기존 코드에 타입이 다르면 위 필드를 포함하도록 보완하라.

---

## 2. accuracy 필터 구현

accuracy가 없는 좌표는 신뢰도 낮음으로 처리한다.

기준:

* accuracy <= 25m: good
* 25m < accuracy <= 50m: usable
* accuracy > 50m: reject

단, accuracy > 50m인 좌표는 바로 마커에 반영하지 말고 이전 보정 위치를 유지하라.

구현 함수 예시:

```ts
function isAcceptableAccuracy(loc: CrewLocation): boolean {
  if (loc.accuracy == null) return false;
  return loc.accuracy <= 50;
}
```

그리고 UI 또는 로그에 rejected reason을 남길 수 있게 하라.

---

## 3. 거리 계산 함수 추가

Haversine 공식을 사용해 두 GPS 좌표 사이 거리를 미터 단위로 계산하는 유틸 함수를 추가하라.

```ts
function distanceMeters(
  a: { lat: number; lng: number },
  b: { lat: number; lng: number }
): number {
  const R = 6371000;
  const toRad = (v: number) => (v * Math.PI) / 180;

  const dLat = toRad(b.lat - a.lat);
  const dLng = toRad(b.lng - a.lng);
  const lat1 = toRad(a.lat);
  const lat2 = toRad(b.lat);

  const h =
    Math.sin(dLat / 2) ** 2 +
    Math.cos(lat1) * Math.cos(lat2) *
    Math.sin(dLng / 2) ** 2;

  return 2 * R * Math.atan2(Math.sqrt(h), Math.sqrt(1 - h));
}
```

## 4. 비정상 점프 제거

크루원이 도보 이동한다고 가정한다.
주기적으로 전달되는 좌표가 물리적으로 불가능한 이동이면 버려라.

기준:

* 도보 최대 속도: 3.5m/s
* 허용 거리 = `3.5 * deltaSeconds + accuracyBuffer`
* accuracyBuffer는 `Math.max(next.accuracy ?? 0, 10)`

예시:

```ts
function isAbnormalJump(prev: CrewLocation, next: CrewLocation): boolean {
  const dt = (next.timestamp - prev.timestamp) / 1000;
  if (dt <= 0) return true;

  const dist = distanceMeters(prev, next);
  const maxSpeed = 3.5;
  const accuracyBuffer = Math.max(next.accuracy ?? 0, 10);
  const allowed = maxSpeed * dt + accuracyBuffer;

  return dist > allowed;
}
```

좌표가 abnormal jump로 판단되면 마커를 이동시키지 말고 이전 위치를 유지하라.

중요:

* 3초 주기로 좌표가 오는데 60m 이동하면 대부분 점프로 판단해야 한다.
* 단, timestamp 간격이 길면 허용 거리가 커져야 한다.

## 5. 최근 좌표 히스토리 관리

crewId별로 최근 좌표 히스토리를 관리하라.

조건:

* crewId별 최대 5개 유지
* 15초보다 오래된 좌표는 제거
* accuracy > 50m 좌표는 히스토리에 넣지 않음
* abnormal jump 좌표도 히스토리에 넣지 않음

예시 구조:

```ts
const crewLocationHistory = new Map<string, CrewLocation[]>();
```

## 6. 최근 좌표 가중 평균

단순 평균이 아니라 accuracy가 좋은 좌표에 더 높은 가중치를 둬라.

```ts
function weightedAverage(points: CrewLocation[]) {
  let sumLat = 0;
  let sumLng = 0;
  let sumWeight = 0;

  for (const p of points) {
    const accuracy = Math.max(p.accuracy ?? 50, 1);
    const weight = 1 / accuracy;

    sumLat += p.lat * weight;
    sumLng += p.lng * weight;
    sumWeight += weight;
  }

  return {
    lat: sumLat / sumWeight,
    lng: sumLng / sumWeight,
  };
}
```

## 7. smoothing / Kalman 유사 필터 추가

crewId별 smoother 상태를 관리하라.

```ts
class LocationSmoother {
  private lat?: number;
  private lng?: number;

  update(loc: CrewLocation) {
    if (this.lat == null || this.lng == null) {
      this.lat = loc.lat;
      this.lng = loc.lng;
      return { lat: this.lat, lng: this.lng };
    }

    let alpha = 0.2;

    if ((loc.accuracy ?? 999) <= 15) alpha = 0.6;
    else if ((loc.accuracy ?? 999) <= 30) alpha = 0.4;
    else if ((loc.accuracy ?? 999) <= 50) alpha = 0.2;

    this.lat = this.lat + alpha * (loc.lat - this.lat);
    this.lng = this.lng + alpha * (loc.lng - this.lng);

    return { lat: this.lat, lng: this.lng };
  }
}
```

```ts
const crewSmoothers = new Map<string, LocationSmoother>();
```

## 8. route snap 구현

크루원이 정해진 경로 또는 route polyline을 따라 이동한다면, 보정된 좌표를 route 위로 스냅하라.

구현 요구:

* route는 `{ lat, lng }[]` 배열
* 현재 좌표에서 가장 가까운 route segment를 찾는다
* 해당 segment 위에 투영한 점을 반환한다
* 원래 좌표와 route 사이 거리가 30m 이하이면 route 위 좌표를 사용
* 30~80m이면 약하게 보정하거나 원본 smoothing 좌표 유지
* 80m 이상이면 경로 이탈로 판단하고 snap하지 않음

함수 형태:

```ts
function snapToRoute(
  point: { lat: number; lng: number },
  route: { lat: number; lng: number }[]
): {
  lat: number;
  lng: number;
  distance: number;
} {
  // 구현 필요
}
```

위도/경도 좌표를 그대로 평면 계산하면 오차가 있을 수 있으므로, 현재 위치 근처에서는 meter 단위 local projection을 사용하거나 근사 계산으로 구현하라.

## 12. 주의사항

* accuracy 필터만으로는 60m 튐을 해결할 수 없다.
* 반드시 abnormal jump 제거와 smoothing을 함께 적용해야 한다.
* 크루원이 정해진 경로를 따라 움직이는 서비스라면 route snap이 가장 중요하다.
* 크루원이 도보인지, 자전거인지, 차량인지에 따라 maxSpeed 값을 설정 가능하게 만들어라.
* 모든 threshold는 상수로 분리하라.

예시 상수:

```ts
const LOCATION_CONFIG = {
  MAX_ACCEPTABLE_ACCURACY_M: 50,
  GOOD_ACCURACY_M: 25,
  MAX_HISTORY_SIZE: 5,

  OFF_ROUTE_DISTANCE_M: 80,
```

## 테스트 케이스

1. accuracy가 80m인 좌표는 reject되어야 한다.
2. accuracy가 20m인 좌표는 accept되어야 한다.
3. 3초 간격으로 60m 이동한 도보 좌표는 abnormal jump로 reject되어야 한다.
4. 3초 간격으로 5m 이동한 도보 좌표는 accept되어야 한다.
5. 최근 5개 이상의 히스토리가 쌓이면 오래된 좌표는 제거되어야 한다.
6. 15초보다 오래된 히스토리는 제거되어야 한다.
7. accuracy가 좋은 좌표가 weighted average에 더 많이 반영되어야 한다.
8. route에서 30m 이내 좌표는 route 위로 snap되어야 한다.
9. route에서 80m 이상 떨어진 좌표는 snap되지 않아야 한다.
10. rejected 좌표가 들어와도 마지막 accepted 위치가 유지되어야 한다.

그리고 테스트 결과를 바탕으로 현재 threshold 값이 너무 엄격하거나 느슨한지 평가하라.
